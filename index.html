What is AIS?

AIS is a utility token purpose-built for AI-driven systems operating on Arbitrum One. Instead of relying on centralized payment processors, AIS enables direct, on-chain settlement for inference tasks, automation pipelines, and usage-based AI billing. Its mission is to provide a frictionless, predictable payment layer for the emerging ecosystem of autonomous agents and AI-native applications.

By leveraging Arbitrum’s low fees and high throughput, AIS is suitable for both micro-transactions and higher-volume workloads, making it a practical medium of exchange for AI-powered products.

Utility and intended use

AIS is engineered for real-world AI payment flows, including:

  • Pay-as-you-go AI inference (per request, per token, or per session).
  • Prepaid credit systems for AI tools and automation workflows.
  • On-chain settlement between AI service providers and users.
  • Integration into dashboards, backends, and metering systems.

AIS does not represent equity, governance rights, or a claim on revenue. It is a pure utility asset designed for operational use within AI-related services.

For developers

AIS follows the ERC‑20 standard on Arbitrum One, making it straightforward to integrate into wallets, dApps, and backend systems. You can use familiar tooling such as ethers.js, web3.js, or wagmi to interact with the contract and build custom payment flows for AI workloads.

Common patterns include mapping AIS transfers to user credits, implementing usage-based billing, and listening to on-chain events to drive automation in your infrastructure.

ethers.js example
import { ethers } from "ethers"; const provider = new ethers.JsonRpcProvider( "https://arb1.arbitrum.io/rpc" ); const ais = new ethers.Contract( "0x1eD25D5f91EfDbC715b4666BfCa19d9573A1ED69", ["function balanceOf(address) view returns (uint256)", "event Transfer(address indexed from, address indexed to, uint256 value)"], provider ); // Read balance const balance = await ais.balanceOf("0xYourAddress"); console.log("AIS balance:", balance.toString()); // Listen for payments ais.on("Transfer", (from, to, value) => { console.log(`AIS Transfer: ${value} from ${from} to ${to}`); });

For integrators

AIS is designed to slot cleanly into existing AI products, dashboards, and automation platforms. A typical integration flow looks like this:

  1. User acquires AIS via Uniswap or any Arbitrum-compatible wallet.
  2. User sends AIS to your service address or payment contract.
  3. Backend listens for AIS Transfer events on Arbitrum.
  4. Credits are issued based on the amount of AIS received.
  5. AI workloads consume credits per request, token, or session.
  6. Usage is tracked and surfaced in your UI or API.
